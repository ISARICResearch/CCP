Data Quality Guidance

DATA QUALITY GUIDANCE

Overview of data quality good practices. These proposals depend on the tools and resources available. They can be implemented at the investigator sites or at a coordinating centre level.

Objective: to ensure the accuracy, completeness, consistency, and reliability of data collected.

DATA QUALITY MANAGEMENT PLAN

A Data Quality Check Management should be provided when setting up the study, describing:

  • Objectives and Scope: Define the objectives of the data quality check plan and specify the scope of data to be covered.
  • Data Quality Dimensions: Identify the dimensions of data quality to be assessed, such as accuracy, completeness, consistency, timeliness, uniqueness, and validity.
  • Data Quality Checks: Specify the checks and tests to be performed, including manual inspections, automated checks, data profiling, and statistical analysis.
  • Frequency and Timing: Determine the frequency and timing of data quality checks (real-time, daily, weekly, monthly).
  • Responsibilities: Assign roles and responsibilities for implementing the data quality check plan.
  • Data Quality Tools: Identify tools and technologies supporting data quality checks.
  • Reporting and Escalation: Establish procedures for reporting and addressing data quality issues.
  • Continuous Improvement: Define processes for monitoring and improving data quality over time.
  • Documentation and Training: Document the plan and provide training to relevant personnel.

The Data Quality Checks Plan can be included in the Data Management Plan.

DATA QUALITY CHECKS

LEVEL 1: Good Questionnaire Design and Programming

  • Real-time field-level validation when entering data.
  • Data validation rules on the eCRF for consistency (e.g., numerical limits, date restrictions, variable consistency).
  • Specific code for missing data (e.g., use “UNK” for unknown data).
  • Avoid free text fields unless necessary; monitor responses to refine question types.
  • Provide guidance for completing eCRF or pCRF to ensure consistency and accuracy.

LEVEL 2: Training

LEVEL 3: Data Quality Checks

1. On-Site Data Entry Check

  • Visual Check: Compare entered data with source documents.
  • Double Data Entry: Two different people enter data for accuracy verification.
  • Scope: Check all or a proportion of entered data (e.g., 5-10% of participants).

2. Centralised Data Programming Checks

  • Inconsistencies and completeness rules: Queries are generated and centrally executed across variables and time periods.
  • Analyzing response distributions: Identifies outliers or irregular patterns.

3. Scope

  • All variables.
  • A proportion of variables.
  • Primary Outcome Data: Recommended to check 100%.
  • Ensure all variables used in the analysis are quality-checked.

4. Frequency

  • After specific visits of certain patients.
  • During study group meetings with dashboards for monitoring outcomes.

Data quality checks should be continuous and occur close to data collection to maintain integrity and validity.

Data Query Management

  • Queries and responses should be saved in the eTMF.
  • Manual Corrections: Use strike-through with initials for paper records.
  • Digital Corrections: The system must have logs to track all corrections (e.g., data history in REDCap).
